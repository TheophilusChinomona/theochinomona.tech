# Add Future Improvement

Add a new improvement entry to `agent-os/specs/future-improvements.md`.

## Instructions

1. First, read the current contents of `agent-os/specs/future-improvements.md`
2. Ask the user for the following details (or extract from context):
   - **Feature Name**: Brief, descriptive title
   - **Priority**: High, Medium, or Low
   - **Complexity**: High, Medium, or Low
   - **Related Spec**: The spec folder name (e.g., `2025-12-29-frontend-scraper-config-refactor`)
   - **Current Behavior**: What happens now
   - **Proposed Improvement**: What should happen
   - **Implementation Notes**: Key steps to implement
   - **Why Not Implemented**: Reason for deferral

3. Add the new entry under the current date heading (create new date heading if needed)

4. Use this format:

```markdown
### [Feature Name]
**Priority:** [High/Medium/Low]  
**Complexity:** [High/Medium/Low]  
**Related Spec:** `[YYYY-MM-DD-spec-name]`

**Current Behavior:**
[Description]

**Proposed Improvement:**
[Description]

**Implementation Notes:**
1. [Step 1]
2. [Step 2]
3. [Step 3]

**Why Not Implemented:**
[Reason]
```

5. Update the "Last Updated" date at the bottom of the file

## Example Usage

User: "Add a future improvement for batch URL validation during crawl"

Response: Add entry with appropriate details based on context from the conversation.
